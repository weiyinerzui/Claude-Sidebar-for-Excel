## 1. 用 Cloudflare Pages 部署前端

大致流程是：GitHub 连接 Cloudflare Pages → 构建 → 自动托管 `dist`。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

1. 先在本地或 GitHub 上确认项目能正常 `npm run build`：  
   ```bash
   npm install
   npm run build
   ```  
   构建产物目录是 `dist/`。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

2. 打开 Cloudflare Dashboard → Pages → Create a project：  
   - 选择 “Connect to Git”。  
   - 选中你的 `Claude-Sidebar-for-Excel` 仓库。  
   - Build command 填：`npm run build`。  
   - Build output directory 填：`dist`。  
   - Node 版本选择 18 或以上（和本地一致）。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

3. 保存并部署，Cloudflare 会自动：  
   - 安装依赖。  
   - 运行 `npm run build`。  
   - 把 `dist` 静态文件托管到一个 HTTPS 域名（例如 `https://your-project.pages.dev`）。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

4. 可选：把自己的域名 CNAME 到这个 Pages 项目，比如 `excel-claude.yourdomain.com`，Cloudflare 会自动签发受信任证书，Excel 能正常访问。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

## 2. 修改 manifest.xml 指向 Cloudflare 域名

部署好后，你会有一个最终访问地址，比如：  

- `https://excel-claude.pages.dev`，或  
- `https://excel-claude.yourdomain.com`（自定义域）。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

接下来在仓库里编辑 `manifest.xml`：  

1. 找到原来指向 dev 的 URL，例如：  
   ```xml
   <SourceLocation DefaultValue="https://localhost:3000/taskpane.html" />
   ```  
 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

2. 替换为 Pages 上的地址，例如：  
   ```xml
   <SourceLocation DefaultValue="https://excel-claude.yourdomain.com/taskpane.html" />
   ```  

3. 如果 manifest 里还有其他 URL（图标、FunctionFile 等）指向 `localhost:3000`，同样改成 Cloudflare 上的域名路径。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

4. 建议保存成单独的生产 manifest，比如 `manifest.prod.xml`，方便区分：  
   - `manifest.dev.xml` → 继续用于本地 `npm run dev` 调试。  
   - `manifest.prod.xml` → 指向 Cloudflare，用于正式给团队用。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

## 3. 团队成员如何使用（本地 Excel + Cloudflare）

每个同事只需两步： [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

1. 在 Excel 中旁加载生产 manifest：  
   - Windows：Insert → Add-ins → My Add-ins → Shared Folder，指向你放 `manifest.prod.xml` 的共享位置（网络盘 / SharePoint / OneDrive）。  
   - macOS：Insert → Add-ins → My Add-ins → Add a Custom Add-in，选中 `manifest.prod.xml`。  

2. 第一次打开侧边栏时，在 Settings 页面输入自己的 Anthropic API Key 并保存。 [github](https://github.com/weiyinerzui/Claude-Sidebar-for-Excel)

之后：  

- Excel 每次都会从 Cloudflare 的 HTTPS 域名加载前端。  
- 你更新代码只要 push 到 GitHub，Cloudflare 自动重新构建 `dist`，团队无感升级。

## 4. 你接下来可以按这个 checklist 走

- 在 Cloudflare Pages 里连上 GitHub 仓库，配置 build 命令和输出目录。  
- 首次部署确认能访问 `taskpane.html`（浏览器打开 Cloudflare 域名）。  
- 修改 manifest，生成 `manifest.prod.xml`，测试在自己 Excel 上能正常加载。  
- 用共享盘 / SharePoint 把 `manifest.prod.xml` 分发给同事。
